Guia Completo: Como Publicar seu Site HTML Online de Graça com GitHub Pages

Este documento detalha o processo, do início ao fim, para colocar um projeto ou arquivo HTML na internet. Ao final, você terá um link público que poderá compartilhar com qualquer pessoa.

Pré-requisitos

  • Uma conta gratuita no GitHub: Se não tiver, crie em github.com.
  • Seu projeto HTML: O arquivo principal do seu site obrigatoriamente deve ser nomeado index.html (tudo em letras minúsculas).

Fase 1: Criar o Repositório (Onde seu projeto vai morar)

  1. Inicie um Novo Repositório: Após fazer login no GitHub, clique no ícone de + no canto superior direito e selecione "New repository".
  2. Configure o Repositório:
    • Repository name: Dê um nome curto e descritivo para o seu projeto (ex: meu-portfolio).
    • Description (Opcional): Adicione uma breve descrição.
    • Public/Private: Selecione "Public". Seu código precisa ser público para o site ser acessível gratuitamente.
    • Initialize: Marque a caixa "Add a README file".
  3. Clique no botão verde "Create repository".

Fase 2: Enviar seus Arquivos para o GitHub

  1. Abra a Ferramenta de Upload: Dentro do seu repositório, clique no botão "Add file" e depois em "Upload files".
  2. Envie os Arquivos: Arraste e solte o seu index.html e todas as pastas e arquivos de apoio (CSS, JS, imagens, etc.) na área indicada.

    3. Lembrete Crucial: O arquivo principal DEVE se chamar index.html.

  3. Confirme o Envio: Role a página para baixo e clique no botão verde "Commit changes". Isso salva os arquivos no seu repositório.

Fase 3: Ativar o GitHub Pages (A Mágica Acontece Aqui)

  1. Acesse as Configurações: Na página principal do seu repositório, clique na aba "Settings" (ícone de engrenagem).
  2. Vá para a Seção "Pages": No menu lateral esquerdo, clique em "Pages".
  3. Configure a Fonte de Publicação:
    • Em "Source", deixe a opção "Deploy from a branch".
    • Em "Branch", selecione o ramo main, a pasta /root, e clique em "Save".

Fase 4: Acessar seu Site Online

  1. Aguarde a Publicação: O GitHub precisa de 1 a 5 minutos para processar seus arquivos e colocar o site no ar.
  2. Encontre seu Link: Permaneça na mesma tela (`Settings` > `Pages`) e atualize a página após alguns minutos. Uma caixa de notificação verde aparecerá no topo com a mensagem: "Your site is live at...".

O formato do link será sempre: https://SEU-NOME-DE-USUARIO.github.io/NOME-DO-SEU-REPOSITORIO/

Como Atualizar seu Site no Futuro?

É muito simples. Basta repetir a Fase 2: envie as novas versões dos arquivos para o mesmo repositório. O GitHub atualizará o site online automaticamente em alguns minutos.

Problemas Comuns e Soluções Rápidas

  • O site aparece em branco: Quase sempre o problema é que o arquivo principal não se chama index.html. Verifique e renomeie se necessário.
  • Fiz uma alteração, mas o site não mudou: Seu navegador pode estar mostrando uma versão antiga. Limpe o cache forçadamente pressionando Ctrl + Shift + R (ou Cmd + Shift + R no Mac).
